核心能力

三个关键环节，覆盖课程论文写作全流程

证据自动提取

自动扫描项目源码，识别技术栈、API 接口、数据库 Schema、测试覆盖率。正文只引用有据可查的内容，绝不编造。

  • 框架自动检测 (Python/Node/Go)
  • API 路由 + DB Schema 提取
  • 测试覆盖率统计

Markdown → DOCX

完整的 GFM Markdown 解析器，支持表格、代码块、LaTeX 公式、Mermaid 图表。一键生成符合学校要求的格式化文档。

  • GFM 表格 + 代码高亮
  • LaTeX 公式渲染
  • Mermaid 图表支持

中文学术格式

预置完整的中文课程论文格式：宋体正文/黑体标题、1.5 倍行距、首行缩进、图表标题规范，开箱即用。

  • 宋体 + Times New Roman 混排
  • 1.5 倍行距 + 首行缩进
  • 图表自动编号排版

三步入门

从初始化项目到生成 DOCX，不到 5 分钟

1

初始化 + 提取证据

在项目根目录运行初始化脚本，编辑论文元信息，自动从源码提取技术证据。

python scripts/init_project.py .
# 编辑 templates/thesis-spec.yaml
python scripts/build_evidence.py . --out paper-context/evidence
2

写 Markdown 论文

按照推荐的五章结构（引言→相关工作→方法→实验→总结），参考章节写作指南完成论文。

# 论文标题
## 摘要
...
## 第1章 引言
...
## 第2章 相关工作
...
## 第3章 方法
...
## 第4章 实验
...
## 第5章 总结与展望
...
## 参考文献
3

生成 DOCX

管理图片映射，一键生成格式规范的 DOCX，附带字数统计和参考文献检查。

python scripts/build_image_map.py --scan paper-output/figures/ paper-output/image-map.json
python scripts/generate_docx.py paper-output/thesis.md paper-output/thesis.docx --image-map paper-output/image-map.json
python scripts/count_words.py paper-output/thesis.md
python scripts/check_references.py paper-output/thesis.md

脚本速览

7 个独立脚本，~800 行 Python，零 Node.js 依赖

脚本行数用途
init_project.py~60一键初始化论文工作区
build_evidence.py193扫描项目源码，提取技术栈/API/Schema/测试证据
generate_docx.py286Markdown → 格式化 DOCX（核心）
build_image_map.py~50将图片目录映射为 JSON，供 DOCX 生成器使用
render_mermaid.py~70渲染 Markdown 中的 Mermaid 图表为 PNG
count_words.py~50按章统计字数
check_references.py~80检查参考文献编号、年份、语言分布

快速开始

在你的项目中使用 AI Course Project Workbench

安装依赖

pip install python-docx PyYAML

初始化工作区

python scripts/init_project.py /path/to/your-project

项目结构（初始化后）

your-project/
├── paper-output/          ← 交付物
│   ├── thesis.md          ← 你写的 Markdown 论文
│   ├── thesis.docx        ← 生成的 DOCX
│   ├── image-map.json     ← 图片映射
│   └── figures/           ← 图表 PNG
├── paper-context/         ← 证据材料
│   ├── evidence/          ← 从源码提取的证据
│   └── literature/        ← 参考文献
└── templates/             ← 论文配置
    ├── thesis-spec.yaml
    └── figure-registry.yaml

格式规格

预置中文学术格式，按学校要求配置（宋体标题 + 1.5 倍行距）

元素字体字号备注
大标题宋体 + Times New Roman22pt (二号)加粗居中
一级标题宋体16pt (三号)加粗
二级标题宋体14pt (四号)加粗
正文宋体 + Times New Roman12pt (小四)1.5 倍行距，首行缩进 24pt
图表标题宋体10.5pt (五号)单倍行距居中
参考文献宋体10.5pt悬挂缩进
代码Consolas9pt
公式Times New Roman12pt居中

需要调整？编辑 scripts/generate_docx.py 中的 build_style_profile() 函数即可。

工程设计

@dataclass 设计

使用 Python dataclass 实现结构化证据提取，类型安全

智能文件扫描

自动忽略 node_modules/、target/、__pycache__ 等构建目录

框架自动检测

通过 package.json、pyproject.toml、go.mod 等标记文件识别技术栈

正则解析器

完整的 GFM Markdown 解析器，支持表格、代码块、LaTeX 公式

OOXML 深度操作

通过 python-docx 直接操作 XML 元素，实现精准中文字体设置

模块化架构

7 个独立脚本，每个都有 argparse CLI，可单独使用或组合调用
